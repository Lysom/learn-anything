### 从2015年到2025年9月的CSS新特性整理

以下是根据可靠来源整理的CSS主要新特性列表，覆盖2015年至2025年9月（当前日期为2025年9月10日）。这些特性主要来自W3C规范、CSS Snapshot和开发者社区总结，如MDN、CSS-Tricks和State of CSS调查。我优先选择了具有广泛浏览器支持和实际影响力的特性，按引入年份大致排序（有些特性在草案阶段逐步成熟）。  

为了清晰呈现，我使用表格格式。表格列包括：**特性名称**、**引入年份**（基于W3C草案或稳定推荐时间）、**作用**、**使用说明**、**代码示例**。注意：浏览器支持可能因版本而异，建议使用Can I Use检查最新兼容性。有些特性在2025年已全浏览器支持。 

| 特性名称 | 引入年份 | 作用 | 使用说明 | 代码示例 |
|----------|----------|------|----------|----------|
| CSS Custom Properties (Variables) | 2015 | 允许定义可复用的变量，用于动态管理样式如颜色、间距。 | 在根元素定义变量，然后在其他地方引用，支持主题切换和响应式设计。 | `:root { --main-color: #06c; } .element { color: var(--main-color); }` |
| Media Queries Level 4 | 2015 | 扩展媒体查询，支持更多设备特性如指针类型、悬停能力。 | 用于检测用户设备（如触摸屏 vs 鼠标），优化UI适应不同输入方式。 | `@media (pointer: fine) { .button { padding: 10px; } }` |
| CSS Grid Layout Level 1 | 2017 | 提供二维布局系统，允许精确控制行和列。 | 用于复杂页面布局，如仪表盘或画廊，比浮动或Flexbox更强大。 | `.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 10px; }` |
| CSS Containment Module Level 1 | 2018 | 优化渲染性能，通过隔离元素防止布局影响外部。 | 在长列表或动态内容中使用，减少重绘，提高页面速度。 | `.item { contain: layout paint; }` |
| CSS Writing Modes Level 4 | 2018 | 支持垂直书写模式和混合方向文本。 | 用于多语言网站，如日文垂直排版或RTL（从右到左）布局。 | `body { writing-mode: vertical-rl; }` |
| :is() Pseudo-class | 2019 | 简化选择器列表，匹配多个选择器之一。 | 减少重复代码，提高可读性，常用于分组伪类。 | `p:is(.intro, .summary) { font-weight: bold; }` |
| aspect-ratio Property | 2020 | 设置元素宽高比，防止图像或视频变形。 | 用于响应式媒体元素，如视频播放器保持16:9比例。 | `.video { aspect-ratio: 16 / 9; }` |
| CSS Color Module Level 5 | 2021 | 引入更多颜色函数，如lab()、lch()，支持广色域。 | 用于精确颜色控制，支持P3显示器和高级调色板。 | `color: lab(50% 40 60);` |
| Container Queries | 2021 | 根据容器大小而不是视口调整样式。 | 用于组件级响应式设计，如卡片在不同父容器中的布局变化。 | `@container (min-width: 400px) { .card { flex-direction: row; } }` |
| :has() Pseudo-class | 2022 | 选择包含特定子元素的父元素（父选择器）。 | 用于条件样式，如如果图片存在则调整布局，无需JS。 | `figure:has(img) { border: 1px solid; }` |
| CSS Nesting | 2022 | 允许在CSS中嵌套选择器，类似于Sass。 | 提高代码组织性，减少重复选择器。 | `.parent { & .child { color: red; } }` |
| Cascade Layers (@layer) | 2022 | 管理样式层级，控制优先级而不依赖选择器权重。 | 用于组织重置、主题、组件层，简化大型项目。 | `@layer base { body { margin: 0; } } @layer theme { body { color: blue; } }` |
| Subgrid | 2022 | 允许子网格继承父网格的轨道。 | 用于嵌套网格布局，确保对齐一致，如表格行。 | `.child { display: grid; grid-template-columns: subgrid; }` |
| Scroll-driven Animations | 2023 | 根据滚动位置驱动动画。 | 用于视差效果或进度指示器，提升交互性。 | `@keyframes parallax { from { transform: translateY(0); } to { transform: translateY(-100px); } } .element { animation: parallax linear both; animation-timeline: scroll(); }` |
| View Transitions | 2023 | 平滑过渡页面或元素变化，如路由切换。 | 用于SPA应用，提供原生过渡动画。 | `::view-transition-old(root) { animation: fade-out 0.5s; }` |
| Anchor Positioning | 2023 | 将元素锚定到另一个元素的位置。 | 用于工具提示或弹出菜单，动态定位。 | `.tooltip { position: absolute; anchor-name: --tooltip; } .anchor { top: anchor(--tooltip bottom); }` |
| text-wrap: balance | 2024 | 平衡多行文本，避免最后一行过短。 | 用于标题或段落，提高可读性。 | `h1 { text-wrap: balance; }` |
| Relative Color Syntax | 2024 | 从现有颜色相对调整，如改变亮度。 | 用于主题变体或渐变，简化颜色计算。 | `color: oklch(from var(--base) l c h / 0.8);` |
| light-dark() Function | 2024 | 根据主题自动切换浅/深颜色。 | 用于暗模式支持，减少媒体查询。 | `background: light-dark(white, black);` |
| Popover API with CSS | 2024 | 原生弹出层，支持锚定和叠加。 | 用于对话框或提示，无需JS框架。 | `[popover] { popover: auto; } button::popovertarget { display: block; }` |
| scrollbar-gutter | 2025 | 预留滚动条空间，防止布局移位。 | 用于滚动容器，确保内容不跳动。 | `.scrollable { scrollbar-gutter: stable; }` |
| ::target-text | 2025 | 高亮锚点链接的目标文本。 | 用于长文档导航，突出跳转位置。 | `::target-text { background: yellow; }` |
| ruby-align & ruby-position | 2025 | 控制Ruby注解的位置和对齐。 | 用于东亚 typography，如汉字注音。 | `ruby { ruby-align: center; ruby-position: over; }` |
| content-visibility | 2025 | 延迟渲染非视口元素，提高性能。 | 用于长页面，减少初始加载时间。 | `.section { content-visibility: auto; }` |
| font-size-adjust | 2025 | 保持字体回退时的可读性。 | 用于自定义字体失败时，调整大小。 | `body { font-size-adjust: 0.5; }` |
| transition-behavior: allow-discrete | 2025 | 允许离散属性（如display）过渡。 | 用于动画显示/隐藏元素。 | `.element { transition: display 0.5s allow-discrete; }` |
| @starting-style | 2025 | 定义元素初始样式，用于进入动画。 | 用于模态框淡入等效果。 | `@starting-style { .modal { opacity: 0; } } .modal { opacity: 1; transition: opacity 0.5s; }` |
| field-sizing | 2025 | 自动调整输入字段大小基于内容。 | 用于文本区动态增长。 | `textarea { field-sizing: content; }` |
| @scope | 2025 | 限定样式作用范围到特定容器。 | 用于模块化CSS，避免全局污染。 | `@scope (.article) to (.figure) { p { margin: 1em; } }` |

这些特性代表了CSS从布局优化到性能提升的演进趋势。2025年重点在于性能（如content-visibility）和交互（如anchor()），许多已获全浏览器支持。 如果需要更详细的浏览器兼容性或特定特性的深入示例，请提供更多细节。
